# 대화(Conversation) 기본 개념

Claude Code는 터미널 기반의 AI 대화 인터페이스입니다. 이 문서에서는 Claude Code와 효과적으로 대화하는 방법을 자세히 설명합니다.

---

## Claude Code 대화의 작동 방식

Claude Code는 일반적인 채팅 앱과 달리 **터미널(Terminal)에서 직접 실행**되는 CLI 도구입니다. 터미널을 열고 `claude` 명령어를 입력하면 대화형 세션이 시작됩니다.

```bash
# Claude Code 대화 시작
claude
```

실행하면 프롬프트(`>`)가 나타나며, 여기에 자연어로 질문이나 요청을 입력할 수 있습니다.

```
> 이 프로젝트의 구조를 설명해줘
```

Claude는 질문에 답하기 위해 필요하다면 파일을 읽거나, 코드를 검색하거나, 명령어를 실행하는 등 다양한 도구를 자동으로 활용합니다.

### 대화의 기본 흐름

1. 사용자가 메시지를 입력합니다.
2. Claude가 메시지를 분석하고, 필요한 도구를 선택합니다.
3. 도구 실행 시 권한이 필요하면 사용자에게 확인을 요청합니다.
4. 결과를 종합하여 응답을 제공합니다.
5. 사용자가 추가 질문이나 요청을 이어갑니다.

## 단일 턴(Single-turn) vs 다중 턴(Multi-turn) 대화

### 단일 턴 대화

하나의 질문에 하나의 답변으로 끝나는 대화입니다. 간단한 질문이나 일회성 작업에 적합합니다.

```bash
# 비대화형 모드로 단일 질문
claude -p "Python에서 리스트를 정렬하는 방법을 알려줘"
```

### 다중 턴 대화

여러 번의 질문과 답변이 이어지는 대화입니다. Claude는 이전 대화 내용을 기억하며, 맥락을 유지합니다.

```
> src/auth.py 파일을 읽어줘

(Claude가 파일을 읽고 내용을 보여줌)

> 이 파일에서 로그인 함수의 보안 취약점이 있어?

(Claude가 이전에 읽은 파일 내용을 바탕으로 분석)

> 발견한 취약점을 수정해줘

(Claude가 맥락을 유지하며 수정 작업 수행)
```

다중 턴 대화에서는 **이전에 논의한 내용을 다시 설명할 필요가 없습니다**. Claude가 대화 전체를 기억하고 있기 때문입니다.

## 컨텍스트(Context)를 효과적으로 활용하기

### 작업 디렉토리의 중요성

Claude Code는 **현재 작업 디렉토리(Working Directory)**를 기준으로 동작합니다. 프로젝트 루트에서 Claude를 시작하면 프로젝트 전체에 접근할 수 있습니다.

```bash
# 프로젝트 루트에서 시작하는 것이 좋습니다
cd ~/my-project
claude
```

### 구체적인 요청하기

Claude에게 요청할 때는 가능한 한 구체적으로 설명하세요.

```
# 나쁜 예 - 너무 모호함
> 코드 고쳐줘

# 좋은 예 - 구체적인 지시
> src/utils/parser.py의 parse_json 함수에서 빈 문자열 입력 시
> ValueError가 발생하는 버그를 수정해줘
```

### 관련 파일 언급하기

Claude가 작업할 파일이나 디렉토리를 명시적으로 언급하면 더 정확한 결과를 얻을 수 있습니다.

```
> src/models/user.py와 src/models/order.py를 참고해서
> src/models/product.py에 Product 모델 클래스를 만들어줘
```

## 파일과 코드를 Claude에게 전달하는 방법

### 방법 1: 대화 중 파일 경로 언급

가장 간단한 방법입니다. Claude가 자동으로 파일을 읽습니다.

```
> src/config.py 파일을 확인하고 데이터베이스 설정을 PostgreSQL로 변경해줘
```

### 방법 2: 코드 직접 붙여넣기

짧은 코드 스니펫은 대화에 직접 붙여넣을 수 있습니다.

```
> 다음 코드에서 버그를 찾아줘:
>
> def calculate_average(numbers):
>     total = sum(numbers)
>     return total / len(numbers)
```

### 방법 3: 파이프(Pipe) 입력 활용

터미널의 파이프 기능을 사용하면 파일 내용이나 명령어 출력을 Claude에게 직접 전달할 수 있습니다.

```bash
# 파일 내용을 Claude에게 전달
cat src/app.py | claude "이 코드를 설명해줘"

# 에러 로그 분석
cat error.log | claude "이 에러의 원인을 분석해줘"

# 명령어 출력 결과 전달
git diff | claude "이 변경사항을 요약해줘"

# 테스트 결과 분석
pytest --tb=short 2>&1 | claude "실패한 테스트를 분석하고 수정 방법을 알려줘"

# 여러 파일을 한 번에 전달
cat src/models/*.py | claude "이 모델들의 관계를 설명해줘"
```

파이프 입력은 `claude -p`와 함께 사용하면 비대화형으로 결과만 출력할 수도 있습니다.

```bash
cat README.md | claude -p "이 문서를 한국어로 번역해줘" > README_ko.md
```

## 비대화형 모드: `claude -p`

`-p` (또는 `--print`) 플래그를 사용하면 대화형 세션 없이 단일 질문에 대한 답변만 받을 수 있습니다. 스크립트나 자동화에 유용합니다.

### 기본 사용법

```bash
# 단일 질문
claude -p "Git에서 마지막 커밋을 취소하는 방법"

# 결과를 파일로 저장
claude -p "FastAPI를 사용한 간단한 REST API 예제 코드를 작성해줘" > api_example.py
```

### 스크립트에서 활용

```bash
#!/bin/bash
# 코드 리뷰 자동화 스크립트
for file in src/*.py; do
    echo "=== $file 리뷰 ==="
    claude -p "$(cat $file)" "이 코드를 리뷰하고 개선점을 알려줘"
    echo ""
done
```

### 출력 형식 지정

```bash
# JSON 형식으로 출력 요청
claude -p "현재 디렉토리의 Python 파일 목록을 JSON 배열로 알려줘" --output-format json

# 스트리밍 출력
claude -p "이 프로젝트의 아키텍처를 설명해줘" --output-format stream-json
```

### 대화형 모드와의 차이점

| 특성 | 대화형 (`claude`) | 비대화형 (`claude -p`) |
|------|-------------------|----------------------|
| 다중 턴 | 가능 | 불가능 (단일 턴만) |
| 권한 요청 | 사용자에게 확인 | 자동 거부 (안전 모드) |
| 세션 유지 | 유지됨 | 없음 |
| 사용 목적 | 대화형 개발 | 스크립트, 자동화 |

## 대화 기록(History)과 `/clear`

### 대화 기록

Claude Code는 대화 내용을 세션 동안 유지합니다. 이를 통해 이전에 논의한 내용을 참조할 수 있습니다.

```
> auth 모듈의 구조를 설명해줘

(Claude가 설명)

> 방금 설명한 구조에서 JWT 토큰 관련 부분을 더 자세히 설명해줘

(Claude가 이전 설명을 기반으로 상세 설명 제공)
```

### `/clear` - 대화 초기화

대화 도중 주제를 완전히 바꾸거나, 컨텍스트 창이 커져서 정리가 필요할 때 `/clear` 명령을 사용합니다.

```
> /clear
```

`/clear`를 사용하면:
- 이전 대화 내용이 모두 삭제됩니다.
- 컨텍스트 창이 완전히 비워집니다.
- 새로운 대화가 시작됩니다.
- 비용이 절감됩니다 (컨텍스트가 작아지므로).

### 언제 `/clear`를 사용해야 할까?

- **주제 전환 시**: 완전히 다른 작업을 시작할 때
- **컨텍스트 오염 시**: 이전 대화 내용이 현재 작업을 혼란스럽게 만들 때
- **비용 관리 시**: 컨텍스트가 너무 커졌을 때
- **오류 누적 시**: Claude가 잘못된 가정을 계속 유지할 때

### 대화 이력 탐색

이전 세션의 대화를 다시 불러올 수도 있습니다.

```bash
# 이전 대화 이어서 진행
claude --continue

# 가장 최근 대화 이어가기
claude -c

# 특정 세션 선택하여 재개
claude --resume
```

## 효과적인 대화를 위한 팁

### 1. 명확한 목표 제시

```
# 나쁜 예
> 코드 개선해줘

# 좋은 예
> src/api/handlers.py의 에러 핸들링을 개선해줘.
> 현재는 모든 예외를 500으로 반환하는데,
> 적절한 HTTP 상태 코드를 반환하도록 수정해줘.
```

### 2. 단계별 접근

큰 작업은 여러 단계로 나누어 요청하세요.

```
> 1단계: 먼저 현재 데이터베이스 스키마를 확인해줘
(확인 후)
> 2단계: users 테이블에 email_verified 컬럼을 추가하는 마이그레이션을 만들어줘
(완료 후)
> 3단계: User 모델에 email_verified 필드를 추가하고 관련 API를 업데이트해줘
```

### 3. 제약 조건 명시

```
> Express.js 미들웨어를 작성해줘.
> 조건:
> - TypeScript로 작성
> - 외부 라이브러리 사용 금지
> - 에러 로깅은 winston 사용
> - 기존 src/middleware/ 디렉토리의 스타일을 따를 것
```

### 4. 예상 결과 설명

```
> 이 함수를 리팩토링해줘.
> 입력: {"name": "Kim", "age": 25}
> 기대 출력: "Kim (25세)"
```

### 5. 피드백 제공

Claude의 결과물이 기대와 다르면 구체적으로 피드백하세요.

```
> 좋은 시작이지만, 몇 가지 수정이 필요해:
> 1. 에러 메시지를 한국어로 변경해줘
> 2. 로깅 레벨을 DEBUG에서 INFO로 변경해줘
> 3. 반환 타입을 Optional[str]에서 str로 변경하고, None 대신 빈 문자열을 반환해줘
```

### 6. CLAUDE.md 활용

프로젝트 루트에 `CLAUDE.md` 파일을 만들어 프로젝트 규칙과 컨텍스트를 지속적으로 제공할 수 있습니다.

```markdown
# CLAUDE.md
- 이 프로젝트는 Python 3.11 + FastAPI를 사용합니다.
- 코드 스타일: Black + isort
- 테스트: pytest 사용
- 모든 API 응답은 표준 응답 포맷을 따릅니다.
```

### 7. 슬래시 명령어 활용

대화 중 유용한 슬래시 명령어들을 활용하세요.

| 명령어 | 설명 |
|--------|------|
| `/clear` | 대화 기록 초기화 |
| `/compact` | 대화 내용 압축 (컨텍스트 절약) |
| `/help` | 도움말 표시 |
| `/cost` | 현재 세션의 토큰 사용량과 비용 확인 |
| `/doctor` | Claude Code 설치 상태 점검 |

## 요약

| 항목 | 설명 |
|------|------|
| 대화 시작 | `claude` 명령어로 대화형 세션 시작 |
| 단일 질문 | `claude -p "질문"` 으로 비대화형 실행 |
| 파이프 입력 | `cat file \| claude "설명해줘"` |
| 대화 이어가기 | `claude --continue` 또는 `claude -c` |
| 대화 초기화 | `/clear` 입력 |
| 대화 압축 | `/compact` 입력 |

효과적인 대화의 핵심은 **구체적이고 명확한 요청**, **단계별 접근**, 그리고 **적절한 컨텍스트 관리**입니다.
